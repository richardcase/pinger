# Implementation Plan: Periodic Ping Monitor

**Branch**: `001-periodic-ping-monitor` | **Date**: 2026-05-27 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/001-periodic-ping-monitor/spec.md`

## Summary

Build a CLI tool (`pinger`) that sends ICMP echo requests to up to 10 configured targets on a
configurable interval, appends each probe result to a daily JSON Lines file, and provides a
`report` subcommand to summarise connectivity history. The tool requires raw socket privileges
(root or `CAP_NET_RAW`), is configured via TOML, and is distributed as cross-compiled binaries
via GoReleaser on GitHub.

## Technical Context

**Language/Version**: Go 1.22+

**Primary Dependencies**:
- `github.com/spf13/cobra` — CLI framework
- `github.com/spf13/viper` — TOML config loading with defaults
- `github.com/go-ping/ping` — ICMP probe execution (privileged raw socket mode)
- `github.com/stretchr/testify` — test assertions

**Storage**: Local filesystem — one `.jsonl` file per calendar day under `data_dir`

**Testing**: `go test` (stdlib) + testify; integration tests behind `//go:build integration` tag

**Target Platform**: Linux/macOS (amd64 and arm64)

**Performance Goals**:
- Probe overhead < 1ms (constitution principle IV)
- Startup to first probe dispatch < 500ms (constitution principle IV)
- Report for 100k records < 1s (SC-004)

**Constraints**:
- Root or `CAP_NET_RAW` required at runtime
- Maximum 10 targets
- Default probe timeout 5s; per-target override supported
- `--duration` accepts Go-style strings (`30s`, `5m`, `1h`, `1h30m`)

**Scale/Scope**: Up to 10 targets; single operator; long-running process

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

### I. Code Quality ✅

- Each internal package (`config`, `probe`, `store`, `report`, `monitor`) has one responsibility.
- golangci-lint (via mise) enforces formatting, error-check, and staticcheck on every PR.
- Cobra/Viper eliminate boilerplate; no dead code or speculative helpers.

### II. Testing Standards ✅

- Unit tests mock the `Prober` interface — no real network required.
- Integration tests (tagged `integration`) send real ICMP to `127.0.0.1` and require root.
- `time.ParseDuration` and JSON marshalling are deterministic — no sleep-based timing in tests.
- Coverage must not regress; each FR has at least one corresponding test case.

### III. User Experience Consistency ✅

- Cobra enforces consistent flag naming and `--help` on all subcommands.
- `pinger report` outputs human-readable table by default; `--json` flag for machine parsing
  (satisfies FR-007 and constitution principle III).
- Error messages follow `error: <what>: <why>. <action>.` format throughout.
- TOML config keys use `snake_case` consistently; no synonyms.

### IV. Performance Requirements ✅

- One goroutine per target; probes dispatched concurrently — sequential polling is a defect.
- Writer goroutine serialises file appends via channel; no lock contention on the hot path.
- Go binary startup is typically < 50ms; well within 500ms budget.
- No unbounded buffers: channel capacity = number of targets; result channel drained per cycle.

**No violations. All gates PASS.**
